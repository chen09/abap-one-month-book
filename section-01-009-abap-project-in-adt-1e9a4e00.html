Chapter 01 - SAP 和 ABAP 基础

9. ABAP project in ADT

  • Chapter 01
  • Section 09
  • self-contained study note

1) 一句话结论

ABAP project in ADT 是 ADT 里连接 ABAP 后端系统的项目入口；本节要学的是：它不是本地源码目录，而是带有 project name、system/client/user/default language 等登录上下文的系统连接，并通过 Project Explorer、Favorite Packages 和权限控制，把开发者带到远端 ABAP repository object 上工作。

2) 到底学什么

上一节 ABAP Development Tools in Eclipse 讲了 ADT 是现代 ABAP 的主要开发工作台。本节把其中最容易被初学者忽略的一层放大：打开 ADT 之后，真正的开发起点不是“新建一个本地文件夹”，而是“新建或打开一个 ABAP project”。SAP Help 对 ABAP project 的定义很直接：它总是代表一个 system connection，并在 SAP system 与 Eclipse-based IDE client 之间起 mediator 的作用。

一个 ABAP project 通常包含几类关键信息。第一是 project name，也就是 Eclipse workspace 里显示的项目名；第二是 system data，例如 system ID、client、user 和认证信息；第三是 default language；第四是可选的 Favorite Packages。SAP Learning 也说明，ABAP project 包含 system、client、user、language 等系统登录参数，开发者必须登录系统才能编辑对象。换句话说，ABAP project 的“项目性”不是来自本地目录结构，而是来自你连接到哪个 ABAP back-end、用哪个 client、哪个 user、哪种 language、以及准备操作哪些 package。

这里有一个对日本项目很重要的细节：ABAP project 的 default language 创建后不能修改。如果开发者一开始用 JA 创建项目，后来又想在同一个项目里改成 EN，通常不能像切 IDE 语言那样简单切换；实际做法是为同一个系统创建另一个 ABAP project。SAP Help 也明确说，同一个 Eclipse workspace 里可以有多个 ABAP projects，甚至可以为同一个系统建立多个项目，用于不同 client 或不同 logon language。

本节还要分清两类项目创建流程。classic ABAP Project 用于 on-premise、SAP S/4HANA Cloud Private Edition 以及其他以 SAP Logon / manual system connection 方式连接的 ABAP 系统。ABAP Cloud Project 则用于 SAP BTP ABAP Environment 和 SAP S/4HANA Cloud Public Edition 3-system landscape（3SL）的开发连接。不要把“cloud”说得太泛：Private Edition 和 on-premise 不是因为名字里有 Cloud 或 S/4HANA 就使用 ABAP Cloud Project。

最后，本节要学一个 practical checklist。打开或创建 ABAP project 时，先确认 system/client、user、default language、Favorite Packages 和基本开发权限。如果连接或编辑失败，不要马上进入安装细节，先回到后文的 canonical 排查顺序，判断问题属于 project type、authentication、authorization，还是后端 ADT 服务/项目环境准备。Day 02 不要求解决所有 setup 问题，但必须知道这些问题属于哪里。

3) 为什么重要

Java/Python 开发者很容易把“项目”理解为本地 working directory。这个经验在 ABAP 里会误导人：本地 Eclipse workspace 只是入口，真正的开发对象在 ABAP back-end repository。ABAP project 连接错了，看到的 package、对象、client 数据、权限、language、transport context 都可能不同。哪怕代码名字相同，DEV 的 client 100、client 200、QAS、Public Edition development system、BTP ABAP Environment service instance，都不是同一个工作上下文。

这个差异会直接影响项目交付。日本 SAP 项目里，开发者经常需要向组长或审查者说明：どのシステム、どのクライアント、どのユーザ、どのパッケージで作業していますか。这个问题不是形式主义。连接到错误 client，可能查不到测试数据；使用管理员用户做开发，可能掩盖权限问题；logon language 错误，可能让 message text、object text 或错误信息不符合项目检查习惯；Favorite Packages 没设好，可能在巨大 System Library 里迷路；权限不足，可能只能 display，不能 activate 或 assign transport。

ABAP project 也是后续所有 ADT 工作流的地基。Repository object 仓库对象、ABAP package 包与子包、Transport request 传输请求 都发生在某个系统连接之下；activation、where-used、ATC 等检查也都继承这个连接上下文。初学者如果跳过这一节，后面会把“我看不到对象”“我不能激活”“为什么 QAS 没变”“为什么 service binding 不能创建”都误判成代码问题。很多时候，问题的第一层根因其实是 project、client、user、language、package 或 authorization。

4) Key mechanism / 核心机制

第一层机制是 workspace 与 project。Eclipse workspace 可以同时放多个 ABAP projects。一个开发者可以在同一个 Eclipse 里连接 DEV、QAS、sandbox、BTP ABAP Environment，也可以为同一个 back-end system 建多个 project，例如 client 不同、language 不同、或登录用户不同。workspace 是本地 IDE 的组织空间；ABAP project 才是进入某个 ABAP back-end 的连接入口。

第二层机制是 project specification。ABAP project 至少要让 ADT 知道 project name、system data、authentication context 和 default language。system data 里通常包括 system ID、client、user 等。default language 会影响消息和 UI 文本，并经由创建时的 logon language 影响新建开发对象的原始语言（original language）。这个设置创建后不能随便改，所以日本项目里经常需要提前决定用 JA 还是 EN，并在必要时创建多个 project。

第三层机制是 classic ABAP Project 与 ABAP Cloud Project。先记住准确边界：classic ABAP Project 适用于 on-premise 和 SAP S/4HANA Cloud Private Edition；ABAP Cloud Project 用于 SAP BTP ABAP Environment 与 SAP S/4HANA Cloud Public Edition 3SL。BTP ABAP Environment 常通过 ABAP service instance URL 或 service key 创建连接，并通过 browser authentication 登录。Public Edition 的 Project Explorer 范围更受限制，通常不会像 classic project 那样暴露完整 System Library。

第四层机制是 Project Explorer。创建 project 后，开发者通常从 Project Explorer 进入 repository。classic project 里可能看到 Favorite Packages、System Library 等入口；cloud project 里对象类型和树结构会更受限制，可能更强调 Favorite Packages、Favorite Objects、Released Objects 等视图。初学者不要一开始就展开整个 System Library，而是把当天课程或项目交付相关 package 加入 Favorite Packages，让工作范围可控。

第五层机制是 Favorite Packages。Favorite Packages 不是装饰项，而是实际工作范围管理。一个 SAP S/4HANA 系统里的 repository 很大，直接从全局树找对象会非常低效。把当前练习包、项目包、POC 包加入 Favorite Packages 后，开发者才能稳定地创建 class、CDS、service binding、package 子对象，也更容易在 code review 或 pair work 中对齐“我们今天改的是哪一包”。

第六层机制是 authorization 与 connection failure。能创建 project 不代表能改对象。Day 02 只需要建立一个 canonical 排查顺序：

ABAP project connection checklist
  1. Project type: classic ABAP Project or ABAP Cloud Project
  2. System/client/user/language: 是否连接到正确环境
  3. Favorite package: 是否能看到目标 package
  4. Authorization: display/change/activate/transport/ATC 等权限是否足够
  5. Backend/service setup: ADT service、service instance URL、service key 或 browser authentication 是否正确

BTP ABAP Environment / Public Edition 里还要注意 browser authentication 当前使用的是 developer user 还是已有 admin session。这里的目的不是教初学者配置系统，而是让他能把问题交给正确的人：Basis、BTP admin、security admin，还是自己修正 ADT project 选择。

第七层机制是 setup boundary。安装 Eclipse/ADT、配置 /sap/bc/adt 等 ICF service、维护 SICF、设置 SNC/SSO、分配标准角色、创建 BTP service key、处理 service key rotation，都很重要，但属于 setup/admin appendix 或项目环境准备工作。本节只给开发者一个判断能力：遇到这些问题时知道该找 Basis、BTP admin、security admin，还是自己在 ADT 里改 project 设置。

5) 日本项目现场说法

  • 常见日语：ABAPプロジェクト、ABAP Cloud Project、システム接続、クライアント、ログオンユーザ、ログオン言語、原語、バックエンドシステム、Project Explorer、お気に入りパッケージ、お気に入り登録、権限不足、接続エラー、システム接続エラー、サービスインスタンスログオンエラー、認証、ブラウザ認証。
  • 说明句：この ABAPプロジェクトは DEV のクライアント 100 に接続しており、ログオンユーザとログオン言語を確認済みです。
  • 说明句：お気に入りパッケージに対象パッケージを登録してから、Repository Object を作成してください。
  • 说明句：S/4HANA Cloud Private Edition の場合は ABAP Cloud Project ではなく、通常の ABAP Project で接続します。
  • 说明句：接続エラーの場合は、プロジェクト種別、ログオンユーザ、権限、サービスインスタンス URL / Service Key を順番に確認します。
  • 使用场景：初回環境セットアップ、開発着手前確認、権限不足の問い合わせ、BTP ABAP Environment 接続確認、Public Edition developer extensibility、移送前の作業環境確認。

  • Official / search term: ABAP project, ABAP Project, ABAP Cloud Project, classic ABAP Project
  • Related terms: system connection, ABAP back-end system, system ID, client, user, password, default language, logon language, original language, Project Explorer, Favorite Packages, System Library, Released Objects, service instance URL, service key, browser authentication, SAP Logon connection, manual system connection
  • Recommended search pattern: ABAP projects SAP Help ADT default language, ABAP project system client user language SAP Learning, ABAP Cloud Project SAP BTP ABAP Environment service key, ABAP Cloud Project S/4HANA Cloud Public Edition 3SL
  • 读官方资料时要区分 logon language 与 original language。前者是登录和界面/消息语言；后者是开发对象文本的原始语言，在项目交付和翻译场景中会被单独检查。

7) Example / 学习例子

假设学习者要在 DEV 系统里继续 Day 02 练习。第一步不是直接创建 class，而是看 ADT 左侧的 Project Explorer：当前 project 名是什么？连接的是哪个 system/client？登录用户是谁？logon language 是 JA 还是 EN？如果教程要求使用某个 package，例如 ZABAP_LEARN_##，就先确认它是否已经加入 Favorite Packages。

第二步检查 project type。粗略记忆是：on-premise / SAP S/4HANA Cloud Private Edition 通常是 classic ABAP Project；SAP BTP ABAP Environment / SAP S/4HANA Cloud Public Edition 3SL 通常按 ABAP Cloud Project 理解。不要用“是不是云”这种粗略判断代替具体环境判断。

第三步做权限观察。尝试打开、编辑、激活一个练习对象；如果失败，不要马上判断为代码问题，而是按第六层机制的 checklist 定位到 project type、登录上下文、authorization 或 service connection。

第四步记录一个工作环境卡片：

System: DEV / S4D
Client: 100
Project type: classic ABAP Project
User: ZDEV_###
Logon language: EN
Favorite package: ZABAP_LEARN_###
Can display/edit/activate: yes / yes / yes
Transport request: DEVK9#####

这个卡片看起来很小，但它能减少大量沟通成本。日本项目里，很多“我这里可以/你那里不行”的问题，最后都要回到这个连接上下文来解释。

8) Counterexample / Failure mode

  • 误区 1：把 ABAP project 当成本地源码目录。修正：它首先是 system connection；source object 保存在 ABAP back-end repository。
  • 误区 2：只看 system，不看 client。修正：同一个 system 的不同 client 可能有不同业务数据、customizing、测试条件和权限。
  • 误区 3：把所有 cloud 场景都用 ABAP Cloud Project。修正：BTP ABAP Environment / SAP S/4HANA Cloud Public Edition 3SL 与 Private Edition / on-premise 的连接方式不同，要按具体系统类型确认。
  • 误区 4：忽视 default language。修正：default language 创建后不能修改；如果需要不同 logon language，创建另一个 project。
  • 误区 5：不设 Favorite Packages，直接在 System Library 里找对象。修正：真实 S/4HANA repository 很大，Favorite Packages 是日常工作范围管理。
  • 误区 6：把权限错误当成代码错误。修正：先确认 display/change/activation/transport/debug/ATC 等权限和角色，再判断代码问题。
  • 误区 7：把 admin 用户当开发用户。修正：特别是 Public Edition / BTP ABAP Environment 场景，要确认 ADT browser authentication 用的是 developer user，而不是已有 admin session。
